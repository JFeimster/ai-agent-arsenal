# Codex Skill Format

Status: draft/proposed

Use this format for Codex-oriented skills that execute file-scoped repo work with validation and PR handoff.

## Required sections

## Task
Define the exact implementation job.

## File scope
List allowed files and explicitly excluded areas.

## Validation
List the checks required before PR.

## Output
Describe the expected code, docs, or review artifact.

## Guardrails
- Preserve unrelated changes.
- Do not delete or move files unless approved.
- Stop at the approved batch boundary.
